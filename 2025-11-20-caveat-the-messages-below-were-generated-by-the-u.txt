● I can help clarify this! The issue is that /resume is not a valid slash 
  command in Claude Code. Here's what's happening:

  How to Resume Sessions

  Sessions in Claude Code are accessed via CLI flags, not slash commands:

  To continue your most recent conversation:
  claude -c

  To resume and select from available sessions:
  claude --resume

  Why You're Seeing "No conversations found"

  Sessions in Claude Code are:

  1. Directory-specific - stored per working directory
  2. Automatically saved for each conversation
  3. Retained for 30 days by default (configurable via cleanupPeriodDays in
  settings)

  If you're not seeing sessions to resume, it's likely because:
  - You're in a different directory than where previous conversations took
  place
  - Previous sessions have been cleaned up (older than 30 days)
  - This is your first conversation in this directory

  Quick Fix

  From your terminal (not inside an active Claude session), run:
  claude -c

  This will continue your most recent conversation in the current directory.

  To Keep Sessions Longer

  Edit ~/.config/claude-code/settings.json and add:
  {
    "cleanupPeriodDays": 90
  }

  Does that help clarify things? Let me know if you'd like help with
  anything else!
